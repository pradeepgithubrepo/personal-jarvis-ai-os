# Supabase V1 Alignment Plan

**Date:** 2026-06-28  
**Phase:** infrastructure Schema Alignment  

---

## 1. Objectives

1. Create all missing tables on remote Supabase instance (`mobile_signals`, `financial_facts`, `salary_events`, `salary_sources`, `merchant_profiles`).
2. Drop legacy tables no longer referenced by the runtime (`todos`).
3. Re-create or adjust constraints and foreign keys to match SQLite models.
4. Establish a single migration script `sql/supabase_v1_alignment.sql` to execute schema changes.

---

## 2. Table Modifications

### Missing Tables (To Create)
* **`mobile_signals`**: Stores incoming raw signals from mobile agents.
* **`financial_facts`**: Stores canonical ledger records generated by the Financial Agent.
* **`salary_events`**: Stores salary occurrences.
* **`salary_sources`**: Stores registered payroll accounts.
* **`merchant_profiles`**: Stores consolidated vendor attributes.

### Legacy Tables (To Drop)
* **`todos`**: Replaced by `todo_items`.

---

## 3. Idempotent Migration Strategy

* The SQL script will utilize `CREATE TABLE IF NOT EXISTS` and conditional DDL constraints.
* Primary keys are mapped using Postgres `UUID` or `BIGSERIAL` to ensure compatibility.
* Indexes are added on all primary lookup keys (e.g. `idx_todo_items_status`, `idx_facts_type`).
